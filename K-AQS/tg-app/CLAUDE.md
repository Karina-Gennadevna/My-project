# CLAUDE.md — Telegram Mini App «K-A-Q-S Индекс управляемости»

## Структура файлов

```
tg-app/
  index.html   ← весь Mini App: HTML + CSS + JS в одном файле
  CLAUDE.md    ← этот файл
```

Всё приложение — один самодостаточный HTML-файл.
Бэкенд (бот) находится в `../bot/`.

---

## Экраны и навигация

```
WELCOME
  └─[кнопка «Начать»]──────────────────→ QUESTION(0)

QUESTION(n)
  ├─[выбор ответа, не последний в зоне]→ QUESTION(n+1)  [fade внутри экрана]
  ├─[выбор ответа, последний в зоне]───→ MILESTONE(zone) [fade]
  └─[BackButton]───────────────────────→ QUESTION(n-1) или WELCOME

MILESTONE(zone)
  ├─[авто 1.5с, не последняя зона]─────→ QUESTION(первый следующей зоны) [fade]
  └─[авто 1.5с, последняя зона]────────→ LOADER [fade]

LOADER
  └─[авто 2.6с]────────────────────────→ RESULTS [slide]

RESULTS
  ├─[MainButton «Получить разбор»]──────→ sendData() → закрытие Mini App
  └─[«Поделиться результатом»]──────────→ копирование / шеринг
```

**Типы переходов:**
- `goTo(name, 'fwd')` — slide влево (вперёд)
- `goTo(name, 'back')` — slide вправо (назад)
- fade-экраны (milestone, loader) — всегда через opacity

## Где менять данные

### Вопросы
Массив `QUESTIONS` в `<script>` — 15 строк, по 3 на зону.
Порядок строго соответствует порядку зон выше.

### Варианты ответов
Массив `ANSWERS` — 4 объекта `{ label, color, score }`.
score: 0, 1, 2, 3 — не менять.

### Тексты рисков
Объект `RISKS` — по ключу ID зоны.
Показывается если балл зоны < 60.

### Рекомендации
Объект `RECS` — по ключу ID зоны, массив из 3 строк.
Показывается первая строка каждой слабой зоны (топ-3).

### Цвета темы
CSS-переменные в `:root` и `body.dark`.
Акцент: `--accent: #2AABEE`.

## Формула расчёта

```
Ответы:   0 (совсем нет) / 1 (иногда) / 2 (в основном) / 3 (да)
Макс:     15 × 3 = 45 баллов

Индекс    = round(сумма всех / 45 × 100)
Зона      = round(сумма 3 ответов / 9 × 100)
```

Цветовые уровни:
- 0–39  → красный  (#FF3B30) «Критический»
- 40–59 → оранжевый (#FF9500) «Низкий»
- 60–74 → жёлтый  (#FFCC00) «Средний»
- 75–89 → зелёный (#34C759) «Хороший»
- 90–100 → яркий зелёный (#30D158) «Высокий»

## Telegram API — что используется

| Метод | Где |
|-------|-----|
| `tg.ready()` | при загрузке |
| `tg.expand()` | при загрузке |
| `tg.colorScheme` | определение тёмной темы |
| `tg.BackButton.show/hide/onClick` | экран вопросов |
| `tg.MainButton.setParams/onClick` | экран результатов |
| `tg.HapticFeedback.impactOccurred('light')` | выбор ответа |
| `tg.HapticFeedback.notificationOccurred('success')` | milestone |
| `tg.sendData(JSON)` | отправка результатов в бот |

**Важно:** `sendData()` работает только если Mini App открыт через
`InlineKeyboardButton` с `web_app`. Не через MenuButton и не прямой ссылкой.

## Тестирование в браузере

Открыть `index.html` напрямую в браузере — всё работает.
Telegram API заменён заглушкой (см. `const tg = window.Telegram?.WebApp || {...}`).

В браузере:
- `BackButton` реагирует на Backspace
- `sendData` пишет в `console.log`
- Тема берётся из `prefers-color-scheme`

## Деплой

1. Запушить в GitHub-репозиторий
2. Задеплоить папку `tg-app/` на Vercel (или любой HTTPS-хостинг)
3. В BotFather: Menu → Web App → указать URL `https://ваш-домен/`
4. В боте: кнопка `InlineKeyboardButton` с `web_app: { url: '...' }`

HTTPS обязателен. Без него Telegram не откроет Mini App.
