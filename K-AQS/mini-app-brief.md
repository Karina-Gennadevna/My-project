# Brief: Mini App «K-A-Q-S Индекс управляемости бизнеса»
_Версия 1.0 | Дата: 2026-03-11 | На основе research.md_

---

## Суть продукта

Telegram Mini App для предпринимателей. 15 вопросов, 5 минут — получаешь индекс управляемости бизнеса 0-100, видишь слабые зоны и топ-3 конкретных шага.

Открывается из бота. Работает как приложение, не как сайт.

---

## Список экранов

```
1. Welcome
2. Вопрос (x15, один экран на вопрос)
3. Milestone — переход между зонами (x5)
4. Прелоадер результатов
5. Результаты
```

Итого: 1 + 15 + 5 + 1 + 1 = 23 состояния, всё в одном HTML-файле.

## Экран 2 — Вопрос

Один экран = один вопрос. Анимированный переход slide left (200ms).

### Что видит пользователь

```
+---------------------------------+
| <- | ########-------  7/15     |  прогресс-бар + счётчик
|    | Финансы · 1 из 3           |  зона + номер в зоне
+---------------------------------+
|                                 |
|  У вас есть регулярная          |
|  финансовая отчётность          |
|  (P&L, cashflow)?               |
|                                 |
+---------------------------------+
| [  Совсем нет                 ] |
|                                 |
| [  Иногда / частично          ] |
|                                 |
| [  В основном да              ] |
|                                 |
| [  Да, полностью              ] |
+---------------------------------+
```

### Элементы
- Прогресс-бар (сверху, цветной, только растёт)
- BackButton Telegram нативный (стрелка влево в шапке)
- Иконка зоны + название + «N из 3»
- Текст вопроса (H2, крупный, 20px semibold)
- 4 карточки ответов (высота min 56px, полная ширина)

### Поведение карточки ответа
- Иконка цвета (красный/оранжевый/жёлтый/зелёный) + текст
- Тап: карточка мгновенно подсвечивается (border акцент + bg с opacity) + чекмарк
- Через 350ms: автоматический переход к следующему вопросу
- HapticFeedback impactOccurred('light') при каждом тапе

### Зоны и иконки
| Зона                  | Иконка | Вопросы |
|-----------------------|--------|---------|
| Финансы               | 💰     | 1-3     |
| Маркетинг             | 📣     | 4-6     |
| Команда               | 👥     | 7-9     |
| Операции              | ⚙️     | 10-12   |
| Контроль собственника | 🎯     | 13-15   |

### Переходы
- Тап на ответ -> следующий вопрос (или Milestone после последнего в зоне)
- BackButton -> предыдущий вопрос (ответ сбрасывается, можно выбрать снова)
- На вопросе 1 BackButton -> Welcome

## Экран 5 — Результаты

Главный экран. Скроллируемый. Максимум визуала.

### Структура сверху вниз

```
+---------------------------------+
|  Индекс управляемости           |
|                                 |
|            67                   |  анимация 0->67 за 800ms
|          из 100                 |
|       Средний уровень           |  статус + цвет
|                                 |
|        [radar chart]            |  SVG 5 осей, анимированный
|                                 |
|  --- Оценка по зонам ---        |
|  🎯 Контроль    30  [###----]   |  от худшей к лучшей
|  ⚙️  Операции   48  [####---]   |  анимированные бары
|  👥 Команда     55  [#####--]   |
|  📣 Маркетинг   65  [######-]   |
|  💰 Финансы     78  [#######]   |
|                                 |
|  --- Главные риски ---          |
|  ⚠️  Бизнес зависит от вас      |
|  ⚠️  Процессы не описаны        |
|  ⚠️  Нет KPI у команды          |
|                                 |
|  --- Что делать сейчас ---      |
|  1. Внедрить weekly-отчёт       |
|  2. Прописать 3 ключевых        |
|     процесса                    |
|  3. Поставить KPI команде       |
|                                 |
|  [ Поделиться результатом ]     |  текстовая ссылка
|                                 |
|  [  Получить разбор в боте  ]   |  MainButton Telegram
+---------------------------------+
```

### Блок «Индекс»
- Цифра 64px, анимация счётчика 0->N за 800ms (requestAnimationFrame)
- Цвет и статус:
  - 0-39  -> «Критический» красный
  - 40-59 -> «Низкий» оранжевый
  - 60-74 -> «Средний» жёлтый
  - 75-89 -> «Хороший» зелёный
  - 90-100 -> «Высокий» насыщенно-зелёный

### Блок «Radar chart»
- SVG, 5 осей: Финансы / Маркетинг / Команда / Операции / Контроль
- Все оси подписаны снаружи
- Заливка: цвет = цвет уровня индекса, opacity 0.25
- Анимация: линии «вырастают» от центра (stroke-dashoffset)

### Блок «Оценка по зонам»
- 5 строк, сортировка от худшей к лучшей
- Иконка + название + цифра + цветной точка-индикатор + анимированный бар
- Бары появляются с stagger 100ms каждый

### Блок «Риски»
- Только зоны с баллом < 60
- Максимум 3 риска
- Тексты привязаны к конкретным ответам пользователя

### Блок «Рекомендации»
- Ровно 3, привязаны к самым слабым зонам
- Формат: конкретное действие + зачем

### CTA
- Текстовая ссылка: «Поделиться результатом»
- MainButton: «Получить разбор в боте»

### Действия
- MainButton -> sendData(JSON) -> бот пишет результаты в чат, Mini App закрывается
- «Поделиться» -> openTelegramLink() с текстом результата

## Telegram API — использование

```javascript
// При загрузке — первым делом
Telegram.WebApp.ready()
Telegram.WebApp.expand()

// BackButton — показываем на экранах вопросов и milestone
Telegram.WebApp.BackButton.show()
Telegram.WebApp.BackButton.onClick(goBack)
// На Welcome и Results — скрываем
Telegram.WebApp.BackButton.hide()

// Haptic при выборе ответа
Telegram.WebApp.HapticFeedback.impactOccurred('light')

// Haptic при завершении зоны
Telegram.WebApp.HapticFeedback.notificationOccurred('success')

// MainButton на экране результатов
Telegram.WebApp.MainButton.setText('Получить разбор в боте')
Telegram.WebApp.MainButton.show()
Telegram.WebApp.MainButton.onClick(sendResults)

// Отправка результатов в бот
Telegram.WebApp.sendData(JSON.stringify({
  index: 67,
  zones: {
    finance: 78,
    marketing: 65,
    team: 55,
    ops: 48,
    control: 30
  },
  risks: ['Контроль собственника', 'Операции', 'Команда'],
  recommendations: [
    'Внедрить weekly-отчёт для команды',
    'Прописать 3 ключевых процесса',
    'Поставить KPI каждому сотруднику'
  ]
}))
```

## Дизайн-система

### Цвета
```css
--bg:           #0D1117;   /* фон */
--surface:      #161B22;   /* карточки, панели */
--border:       #30363D;   /* обводки неактивных элементов */
--text:         #F0F6FC;   /* основной текст */
--muted:        #8B949E;   /* второстепенный текст */
--accent:       #F97316;   /* оранжевый акцент */

--red:          #EF4444;   /* индекс 0-39 */
--orange:       #F97316;   /* индекс 40-59 */
--yellow:       #EAB308;   /* индекс 60-74 */
--green:        #22C55E;   /* индекс 75-89 */
--teal:         #10B981;   /* индекс 90-100 */
```

### Типографика
- Шрифт: Inter (Google Fonts) + fallback system-ui, sans-serif
- H1 welcome: 24px, 700
- H2 вопрос: 20px, 600
- Карточка ответа: 16px, 400
- Подписи (зона, счётчик): 13px, --muted

### Карточки ответов
- Высота: min-height 56px
- Padding: 14px 16px
- Border-radius: 12px
- Gap между карточками: 10px
- Неактивная: border 1.5px solid var(--border), bg var(--surface)
- Выбранная: border 1.5px solid var(--accent), bg rgba(249,115,22,0.12), + чекмарк справа

### Анимации
- Slide между вопросами: translateX(-30px) + opacity 0->1, 200ms ease-out
- Прогресс-бар fill: width transition 300ms ease
- Radar chart линии: stroke-dashoffset, 600ms ease-out, stagger 80ms на ось
- Счётчик индекса: requestAnimationFrame, 800ms, cubic-bezier easeOut
- Бары зон: width 0->N%, 500ms ease-out, stagger 100ms

## Поток навигации (State Machine)

```
WELCOME
  +--> [Начать] --> QUESTION(1)

QUESTION(n)
  +--> [ответ] + [n не последний в зоне]  --> QUESTION(n+1)
  +--> [ответ] + [n последний в зоне]     --> MILESTONE(zone)
  +--> [BackButton]                        --> QUESTION(n-1) или WELCOME

MILESTONE(zone)
  +--> [авто 1.5с] + [не последняя зона]  --> QUESTION(первый следующей зоны)
  +--> [авто 1.5с] + [последняя зона]     --> LOADER

LOADER
  +--> [авто 2.5с] --> RESULTS

RESULTS
  +--> [MainButton]    --> sendData() --> закрытие Mini App
  +--> [Поделиться]   --> openTelegramLink()
```

## Формула расчёта индекса

```
Ответы: 0 (совсем нет) / 1 (иногда) / 2 (в основном) / 3 (да, полностью)
Максимум: 15 вопросов x 3 = 45 баллов

Общий индекс = Math.round((сумма всех ответов / 45) * 100)
Индекс зоны  = Math.round((сумма 3 ответов зоны / 9) * 100)
```

Риски — зоны с индексом < 60, сортируем по возрастанию.
Рекомендации — берём из заранее написанного банка по слабым зонам.

## Банк рекомендаций (по зонам)

### Финансы (если < 60)
- Внедрить ежемесячный P&L-отчёт — знать прибыль, не только выручку
- Посчитать юнит-экономику: стоимость привлечения vs средний чек
- Сформировать финансовую подушку на 2 месяца операционных расходов

### Маркетинг (если < 60)
- Провести анализ источников лучших клиентов за последние 6 месяцев
- Описать воронку привлечения: от касания до первой оплаты
- Начать считать CAC хотя бы по одному каналу

### Команда (если < 60)
- Прописать KPI для каждой роли: 2-3 метрики, еженедельный отчёт
- Задокументировать 3 ключевых процесса (не держать в голове)
- Провести тест: что произойдёт, если вы уйдёте на 2 недели?

### Операции (если < 60)
- Описать стандартный цикл выполнения услуги / производства
- Добавить точку контроля качества до финальной сдачи
- Определить 3 решения, которые команда принимает без вас

### Контроль собственника (если < 60)
- Ввести еженедельный управленческий отчёт (10 минут в пятницу)
- Выписать все задачи, которые только вы делаете — делегировать одну
- Запланировать 2 недели без операционного участия через 3 месяца

## Чего НЕ будет в v1.0

| Что                           | Почему                              |
|-------------------------------|-------------------------------------|
| Регистрация / email-гейт      | Трение убивает конверсию            |
| История результатов           | Нужна БД -> v2                      |
| Ветвление вопросов            | Усложняет код и тестирование        |
| Сегментация по отрасли        | Мало данных на старте -> v2         |
| PDF-отчёт                     | Нужен backend -> v2                 |
| Сравнение с другими           | Нет данных на старте                |
| Платный контент               | v1 — бесплатный лид-магнит          |
| Онбординг (2+ экрана до теста)| Welcome достаточно                  |
