# K-A-Q-S™ Platform — Паспорт проекта
_Последнее обновление: 2026-03-03_

---

## Название проекта
**K-A-Q-S™ Platform** — платформа оценки ИИ-зрелости бизнеса

---

## Что это

Веб-платформа, которая за 20 минут показывает бизнесу: насколько системно у них выстроен AI, где дыры, и что конкретно делать дальше. Компания отвечает на 36 вопросов по авторской методологии K-A-Q-S — и получает балл по 4 модулям, уровень зрелости и роадмап на 90 дней. Результат скачивается как PDF-отчёт — готовый документ для команды или совета директоров.

---

## Для кого

**Основная аудитория:** руководители компаний 20–500 человек — CEO, COO, CDO.

Конкретнее: те, кто уже начал внедрять AI (прикрутили пару инструментов, может даже наняли AI-специалиста), но ощущают, что системы нет. Не технари — им нужны выводы, а не технический жаргон.

**Вторая аудитория:** AI-консультанты и агентства, которым нужен готовый инструмент для диагностики клиентов.

---

## Какую проблему решает

Бизнесы внедряют AI хаотично: один инструмент здесь, другой там, без архитектуры и контроля. Они не знают:
- на каком уровне зрелости находятся
- какие риски критичны прямо сейчас
- с чего начать, чтобы не переделывать потом

Существующие решения (IBM, Gartner, Cisco) либо стоят $30k+, либо бесплатны, но заточены под продажу чужого продукта. На русском языке независимого инструмента нет вообще.

**Факт:** По данным Cisco AI Readiness Index 2025 — только 13% компаний получают измеримый ROI от AI. Остальные 87% не знают, где они находятся и что делать.

## Методология K-A-Q-S™

Авторская система оценки по 4 модулям:

| Модуль | Суть |
|--------|------|
| **K** — Knowledge Architecture | Архитектура знаний: есть ли понимание структуры бизнеса и его процессов |
| **A** — AI Integration Layer | AI встроен в систему или прикручен поверх |
| **Q** — Quality Control Framework | Контроль качества: тестирование, риски, отказоустойчивость |
| **S** — Scalability Design | Готов ли бизнес расти без переписывания всего с нуля |

Каждый модуль — до 30 баллов. Максимум: 120. Результат: 5 уровней зрелости от «Хаоса» до «Лидерства».

## Технологии

| Слой | Выбор | Обоснование |
|------|-------|-------------|
| Frontend + Backend | Next.js 14 + TypeScript | Один стек, SSR, уже написан |
| База данных | Supabase (PostgreSQL) + Prisma | Managed, бесплатный tier, быстрый старт |
| Хостинг | Vercel | Родной для Next.js, деплой за минуту |
| Auth | JWT (jose + bcryptjs) | Уже реализован, HTTP-only cookie |
| Email | Resend | 100 писем/день бесплатно, современный API |
| PDF | window.print() + CSS | Достаточно для MVP, без зависимостей |
| Оплата RU | ЮKassa | Единственный рабочий вариант для России |
| Оплата INT | Paddle | Merchant of Record, меньше налоговых проблем |
| Аналитика | PostHog Cloud | До 1M событий бесплатно, product analytics |

## Чего НЕ будет на старте

- **Мобильного приложения** — сначала web, адаптивный дизайн
- **OAuth / входа через Google** — достаточно email + пароль
- **Отраслевого бенчмарка** — нужно накопить минимум 50 оценок
- **AI-ассистента в чате** — усложняет MVP без проверенной ценности
- **Командного доступа** — это v1.1, не v1.0
- **White-label** — только после первых 10+ платящих клиентов
- **Многоязычности** — сначала только русский
- **Кастомных вопросов** — стандартный опросник для всех

## Первые шаги

1. **Установить Node.js** → `npm install` в папке `kaqs-platform/`
2. **Создать проект в Supabase** → скопировать connection string в `.env.local`
3. **Запустить миграцию БД** → `npx prisma db push`
4. **Задеплоить на Vercel** → подключить репозиторий, добавить env-переменные
5. **Проверить полный флоу** → регистрация → опросник → результаты → PDF
6. **Подключить ЮKassa** → pay-per-report за 4 900 ₽
7. **Опубликовать первый пост в TG/LinkedIn** → «Пройди бесплатную оценку»
8. **Получить 5 первых оценок** от знакомых → собрать фидбэк

## Критический взгляд — что упущено

Просмотрела документ. Вот что стоит проработать:

**1. Нет определения «готового MVP»**
Когда считаем, что MVP запущен? Нужен чёткий DoD: например, «10 человек прошли оценку end-to-end без помощи».

**2. Цены не проверены рынком**
4 900 ₽, 7 900 ₽, 19 900 ₽ — это гипотезы. До подключения оплаты стоит спросить 5–10 потенциальных клиентов: «Сколько вы готовы заплатить за такой отчёт?»

**3. Нет плана получения первых пользователей**
«TG-канал и LinkedIn» — слишком общо. Нужен конкретный список: кому написать лично, в каких чатах анонсировать, какой оффер.

**4. Качество вопросов не проверено**
36 вопросов написаны — но пользователи могут не понимать формулировки. Нужен тест с 3–5 людьми вживую.

**5. Нет правовой базы**
- Политика конфиденциальности (данные пользователей хранятся в Supabase)
- Пользовательское соглашение
- Защита товарного знака K-A-Q-S™

**6. Непонятно, что получает пользователь бесплатно**
Freemium-граница не определена: вся оценка бесплатно или только первые 2 модуля? Это влияет на конверсию.

**7. Нет метрик успеха**
Что означает «платформа работает»? Например: 100 оценок за первый месяц, конверсия в платный отчёт 15%, NPS > 40.
