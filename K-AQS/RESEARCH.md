# Research: Mini App «Индекс управляемости бизнеса»
_Дата: 2026-03-11_

---

## 1. Аналоги и конкуренты

### Telegram Mini Apps (диагностика / опросы)

| Название | Что делает | Сильно | Слабо |
|---|---|---|---|
| **Бизнес-тест боты** (общий класс) | Серия вопросов → результат в чате | Просто, без установки | Нет визуализации, текстовые ответы |
| **Quiz боты** (Typebot, Questionbot) | Опросники с ветвлением | Гибкость вопросов | Нет красивого результата, нет радара |
| **Notion-style диагностики** в Mini App | Чеклисты, самооценка | Удобный интерфейс | Нет скоринга, нет динамики |
| **HR-тесты в Mini App** (оценка команд) | Психотипы, совместимость | Геймификация | Не про бизнес-управляемость |

### Веб-сервисы (прямые конкуренты)

| Название | Что делает | Сильно | Слабо |
|---|---|---|---|
| **ScoreApp** | Quiz-воронка → лид + результат | Красивые результаты, email-захват | Дорого, только английский |
| **Typeform** | Опросники с условиями | Плавный UX, один вопрос за раз | Нет готового скоринга |
| **McKinsey OHI** | Индекс здоровья организации | Авторитетность, детальность | Только Enterprise, сложно |
| **Gallup Q12** | Вовлечённость команды | 12 вопросов, простота | Только про команду, нет финансов |

**Вывод:** нет готового русскоязычного Telegram Mini App с визуализацией индекса управляемости для малого бизнеса. Ниша свободна.

## 2. Стандартные экраны в подобных приложениях

### Обязательные

```
1. Welcome / Splash
   — Название и суть за 2 строки
   — Что получит пользователь (конкретно)
   — Сколько времени займёт
   — CTA-кнопка «Начать»

2. Вопросы (основной экран)
   — Прогресс-бар вверху (обязателен)
   — Название зоны + номер вопроса в зоне
   — Текст вопроса (крупно)
   — 4 варианта ответа — карточки
   — Кнопка «Назад»

3. Переход между зонами
   — «Блок "Финансы" пройден ✓» — мотивирует продолжать
   — Анимация, 1-2 сек

4. Прелоадер результатов
   — «Анализируем данные...» 2-3 сек
   — Результат воспринимается серьёзнее

5. Результаты (главный экран)
   — Общий индекс (большая цифра + анимация счётчика)
   — Radar chart по 5 осям с анимацией
   — Оценка по каждой зоне (цвет + балл)
   — Топ-3 риска
   — Топ-3 рекомендации
   — MainButton «Получить результат в бот»
```

### Опциональные
- Email-захват перед результатами (конверсия выше, но снижает прохождение)
- Детальный отчёт (второй уровень)
- Кнопка «Пройти снова»

---

## 3. Сильные стороны лучших решений

- **Один вопрос на экране** — не пугает объёмом, фокус на одном
- **Прогресс всегда виден** — «Вопрос 7 из 15» или прогресс-бар
- **Цвет реагирует на ответ** — мгновенный фидбек при выборе
- **Зоны разделены визуально** — пользователь понимает структуру
- **Результат вызывает «вау»** — большая цифра + анимация радара
- **Конкретность рекомендаций** — не «улучшите процессы», а «внедрите weekly-отчёт»
- **Telegram Native кнопки** — MainButton снизу вместо кастомных

---

## 4. Что раздражает пользователей

- Больше 20 вопросов — бросают на середине
- Нет объяснения что будет в конце
- Нет кнопки «Назад» — раздражает сильно
- Банальные результаты без конкретики
- Нет визуализации — только текстовый результат
- Нет мотивации завершить — не видно «осталось 3 вопроса»

---

## 5. Что сделаем лучше конкурентов

| Проблема у других | Наше решение |
|---|---|
| Нет визуализации | Animated radar chart + счётчик индекса |
| Банальные рекомендации | Привязаны к конкретным ответам пользователя |
| Нет Telegram-нативности | MainButton, HapticFeedback, themeParams |
| Нет мотивации продолжать | Мини-победа между зонами + прогресс |
| Нет прелоадера | 2 сек «анализируем» — доверие к результату |
| Только текст в боте | Красивое сообщение с эмодзи и структурой |

## 6. Обязательные UI-элементы

### Навигация
- Прогресс-бар (верх экрана, цветной)
- Кнопка «Назад» (←)
- Индикатор зоны («Финансы · 2/3»)

### Карточки ответов
- 4 варианта, одна карточка = один ответ
- Выбранная — подсвечивается цветом + чекмарк
- Автопереход через 300ms или кнопка «Далее»

### Экран результатов
- Индекс 0–100: крупно, анимация счётчика
- Цвет по уровню: красный / оранжевый / жёлтый / зелёный
- Radar chart по 5 осям
- Карточки зон: иконка + название + балл
- Риски: топ-3 с иконкой ⚠️
- Рекомендации: топ-3, конкретные

### Telegram API
- `Telegram.WebApp.ready()` — сразу при загрузке
- `Telegram.WebApp.expand()` — весь экран
- `Telegram.WebApp.MainButton` — кнопка снизу
- `Telegram.WebApp.HapticFeedback` — вибрация при выборе
- `Telegram.WebApp.themeParams` — светлая/тёмная тема
- `Telegram.WebApp.sendData()` — результаты в бот

## 7. Вопросы для опроса (финальный черновик)

**Формат ответов (4 варианта, 0–3 балла):**
- 🔴 Совсем нет (0)
- 🟠 Иногда / частично (1)
- 🟡 В основном да (2)
- 🟢 Да, полностью (3)

### Финансы
1. У вас есть регулярная финансовая отчётность (P&L, cashflow)?
2. Вы знаете вашу юнит-экономику (стоимость клиента, маржу)?
3. Есть финансовая подушка или лимит на 2+ месяца расходов?

### Маркетинг
4. Вы знаете, откуда приходят ваши лучшие клиенты?
5. Есть повторяющийся процесс привлечения клиентов?
6. Вы отслеживаете стоимость привлечения клиента (CAC)?

### Команда
7. Сотрудники знают свои KPI и отчитываются по ним?
8. В команде есть документированные процессы (не только в голове)?
9. Если ключевой сотрудник уйдёт — бизнес продолжит работать?

### Операционные процессы
10. Есть понятный цикл производства / оказания услуги?
11. Вы замеряете качество на каждом этапе?
12. Операционные решения принимаются без вашего участия?

### Контроль собственника
13. Вы тратите менее 30% времени на «тушение пожаров»?
14. Есть управленческий дашборд или еженедельный отчёт?
15. Можете уйти в отпуск на 2 недели без ущерба для бизнеса?

### Расчёт
- Максимум: 15 × 3 = 45 баллов
- Индекс = (сумма / 45) × 100
- По зонам: (сумма по зоне / 9) × 100

## 11. Критические ошибки которых избегаем

1. Нет индикации длины на входе → показываем «15 вопросов · 5 минут» на welcome
2. Прогресс-бар идёт назад → он только растёт
3. Результат без интерпретации → каждый индекс имеет статус + объяснение
4. 5+ CTA на экране результата → один MainButton + одна вторичная
5. Одинаковый вес рекомендаций → топ-3 приоритизированных, с привязкой к слабым зонам
6. Нет кнопки «Назад» → всегда есть

## 12. Инсайты от ScoreApp, McKinsey OHI, Growthwheel

### Что делают лучшие продукты

| Паттерн | Продукт | Что берём |
|---|---|---|
| Большой скор + gauge-chart (спидометр) | ScoreApp | Анимированный счётчик + цветовой индикатор |
| Радар по 9 измерениям | McKinsey OHI | Радар по 5 нашим зонам |
| «Вы лучше 68% компаний» | Gallup, McKinsey | Добавим в результат сравнение с медианой |
| Сегментация на старте | Outgrow | Первый вопрос: размер команды → адаптация рекомендаций |
| «Вы почти у цели» на 75% | Typeform | Milestone «Финишная прямая — осталось 4 вопроса» |
| Топ-3 приоритета, не список из 20 | Все лучшие | Строго 3 рекомендации, привязанные к слабым зонам |

### Ключевые цифры (из данных Typeform 2024)
- **Completion rate без прогресс-бара** падает на 20–30%
- **60–70%** прохождений с телефона → mobile-first обязателен
- **Email до результата** — фейковые адреса или уход, работает только после preview результата

### Главный инсайт
> Пользователи помнят не вопросы, а экран результата.
> Вложить максимум усилий туда, а не в формулировки вопросов.

### Структура экрана результатов (синтез лучших практик)
```
1. Имя / заголовок диагностики
2. Большой индекс: 67/100 + статус («Средний уровень»)
3. Radar chart по 5 зонам
4. Бары по зонам — от худшей к лучшей
5. Топ-3 приоритета: проблема → действие
6. Один CTA (MainButton Telegram)
7. Вторичное: «Поделиться результатом»
```

## 13. Экспертная оценка

### Точка зрения 1: UX-дизайнер

#### Типичные ошибки новичков в quiz-приложениях

1. **Welcome без конкретики** — пишут «диагностика бизнеса» и сразу кнопка «Начать». Пользователь не знает, что получит. Нужно: «Узнайте индекс управляемости своего бизнеса — 15 вопросов, 5 минут, результат сразу».

2. **Прогресс-бар врёт или отсутствует** — самая частая причина брошенного опроса. Если бар не двигается заметно — пользователь думает, что застрял.

3. **Карточки ответов слишком мелкие** — на мобиле tap target меньше 44px превращается в лотерею. Промахнулся — засчитался не тот ответ, нет кнопки «Назад» — всё, уходит.

4. **Переход между вопросами без анимации** — экран «мигает», ощущение дешёвого продукта. Даже 200ms slide влево меняет восприятие.

5. **Экран результата — просто текст** — пользователь ожидает «вау», а получает список из 10 пунктов одинаковым шрифтом. Визуализация = доверие.

6. **Нет milestone между зонами** — 15 вопросов подряд утомляют. Экран «Финансы пройдены ✓» даёт передышку и мотивирует продолжить.

7. **Слишком умные вопросы** — «Оцените степень формализации управленческих процессов». Предприниматель закрывает. Нужен живой язык: «Если ключевой сотрудник уйдёт — бизнес продолжит работать?»

#### Что обязательно должно быть

- Прогресс-бар + «зона N из 5» + «осталось N вопросов» — тройная мотивация
- Автопереход с задержкой 300–400ms (не сразу — случайные тапы)
- Анимация выбора карточки: цвет меняется мгновенно, потом переход
- Прелоадер 2–3 сек перед результатом: «Анализируем ваши ответы...»
- Счётчик индекса: анимация от 0 до финального значения (~800ms)
- Radar chart с анимацией появления осей

#### Что бесит пользователей

- Нет кнопки «Назад» — бесит сильнее всего остального
- Автопереход без задержки — случайный тап на неправильный ответ, и не вернуться
- Одинаковые рекомендации всем — сразу видно, что «шаблон»
- Результат без объяснения: «67 баллов» — и что? Нужен статус + интерпретация

### Точка зрения 2: Маркетолог

#### Что заставит пройти диагностику

1. **Заголовок бьёт в боль** — не «индекс управляемости», а «Узнайте, почему ваш бизнес зависит лично от вас — и как это исправить». Страх потери > желание выгоды.

2. **Конкретное обещание результата** — «получите индекс 0–100, топ-3 риска и план на ближайший месяц». Абстрактная «диагностика» не конвертит.

3. **Социальное доказательство на welcome** — «Уже прошли 847 предпринимателей» (даже если цифра небольшая — она работает).

4. **Срочность/эксклюзивность** — «Бесплатно сейчас» или «Первые 100 участников получают разбор результатов».

#### Элементы, повышающие конверсию в действие после результата

1. **Персонализированный CTA** — не «Записаться на консультацию», а «Разберём ваши 3 слабых зоны за 30 минут». Конкретно, про пользователя.

2. **Якорь «вы хуже X% предпринимателей»** — если индекс низкий, фраза «по нашим данным, 65% предпринимателей в вашей нише управляют бизнесом лучше» создаёт мотивацию действовать.

3. **Триггер «первый шаг уже сделан»** — «Вы уже выявили слабые зоны. Следующий шаг — план устранения». Психология sunk cost работает.

4. **Результат в бот = повторный контакт** — когда результат приходит в Telegram-чат, бот может написать через 3 дня: «Как продвигаетесь с командой? Мы подготовили материал».

5. **Шеринг как вирусный механик** — «Поделиться результатом» с кнопкой → друг видит чужой индекс → проходит сам. Конкуренция между предпринимателями работает.

6. **Нельзя показывать email-гейт до результата** — только после, когда пользователь уже вложил 5 минут и хочет получить полный отчёт.

#### Структура конверсионного CTA на экране результатов

```
[Основной] Telegram MainButton:
"Получить полный разбор → в бот"

[Вторичный] текстовая ссылка:
"Поделиться результатом с партнёром"
```

Больше ничего. Два действия максимум.

### Точка зрения 3: Telegram-разработчик

#### Ограничения Telegram Mini App

1. **`sendData()` работает только если Mini App открыт через KeyboardButton** — если открыт через MenuButton или прямую ссылку, `sendData()` выбросит ошибку. Решение: всегда открывать через `InlineKeyboardButton` с `web_app`.

2. **Размер данных `sendData()`** — максимум 4096 байт. JSON с результатами (15 ответов + индекс + зоны) — ~500 байт. Влезаем с запасом.

3. **HTTPS обязателен** — localhost не работает в реальном Telegram. Для разработки: ngrok или Vercel Preview.

4. **Высота окна меняется** — когда появляется клавиатура (если есть input-поля), высота Mini App сжимается. Избегаем `position: fixed` снизу без учёта `viewportHeight`. Решение: `Telegram.WebApp.onEvent('viewportChanged', handler)`.

5. **Тема подхватывается автоматически** — `themeParams.bg_color`, `text_color`, `button_color`. Не задаём цвета жёстко — используем CSS-переменные из Telegram или фолбэк для нашей тёмной темы.

6. **MainButton — одна кнопка** — нельзя поставить две MainButton. Всё остальное — кастомные элементы внутри страницы.

7. **BackButton** — `Telegram.WebApp.BackButton` — нативная кнопка «Назад» в шапке Telegram. Лучше использовать её вместо кастомной, это нативно и привычно.

#### Что работает хорошо

- **HapticFeedback** — `impactOccurred('light')` при выборе ответа, `notificationOccurred('success')` при финале. Пользователи замечают и любят.
- **`expand()`** — вызываем сразу при загрузке, иначе Mini App открывается в половину экрана.
- **CSS-анимации** — работают отлично, Canvas тоже. Используем для radar chart.
- **`Telegram.WebApp.ready()`** — убирает лоадер Telegram, вызываем как можно раньше.
- **Цвета темы** — `var(--tg-theme-bg-color)` автоматически адаптирует под светлую/тёмную тему пользователя.

#### Что лучше не делать

- **Не использовать `alert()`/`confirm()`** — в Mini App они заблокированы. Только кастомные модалки.
- **Не делать `position: fixed` bottom без `safe-area-inset`** — на iPhone X+ кнопка уходит под home indicator.
- **Не полагаться на `localStorage` между сессиями** — Mini App может быть закрыт и данные потеряются. Для сохранения результатов — только через бот.
- **Не делать тяжёлые анимации на старте** — Telegram открывает Mini App в WebView, первые 300ms часто лагают. Анимации запускать после `ready()`.
- **Не использовать внешние шрифты без фолбэка** — Google Fonts грузятся ~200–400ms, без фолбэка будет FOUT (мигание текста).

#### Архитектура бота (минимальная)

```
Bot (telegraf / node-telegram-bot-api)
  ├── /start → приветствие + кнопка "Открыть диагностику" (web_app)
  ├── on('web_app_data') → получаем JSON результатов
  │     ├── парсим индекс, зоны, ответы
  │     └── отправляем красивое сообщение с результатами
  └── (опционально) сохраняем в БД для истории
```
