## 1. Цель сайта и целевая аудитория

### Цель

Лендинг-портфолио Карины — основателя системы K-A-Q-S™ и архитектора управляемого бизнеса.

**Главная задача:** Конвертировать посетителя в заявку на консультацию или в участника бесплатной диагностики на платформе K-A-Q-S™.

**Вторичные задачи:**
- Объяснить методологию K-A-Q-S™ без корпоративного жаргона
- Сформировать доверие через кейсы, цифры и личный опыт
- Занять позицию независимого эксперта (не продаём чужой продукт)
- Дать ссылку на платформу диагностики как основной продукт

### Целевая аудитория

**Сегмент А — Руководитель ("Хочу понять, где мы стоим")**
- CEO / COO компании 50–500 человек
- Триггер: конкурент внедрил ИИ или автоматизацию, нужно понять что делать
- Ожидание: «Скажите мне что делать, а не что такое ИИ»
- Боли: нет времени разбираться в технологиях, непонятна ROI, боятся ошибиться

**Сегмент Б — Стратег ("Нужна база для совета директоров")**
- CDO / CTO, выступает перед советом или инвесторами
- Ожидание: авторитетный документ с независимой методологией
- Боли: внутренние оценки не воспринимаются как объективные

**Сегмент В — Консультант ("Оцениваю клиентов")**
- AI-консультант, бизнес-аналитик, партнёр акселератора
- Ожидание: инструмент для диагностики клиентов, возможно white-label
- Боли: нет готового инструмента, создавать с нуля дорого

Сдержанная уверенность. Не агрессивный маркетинг, не тёплое «коучинговое». Технологично, но человечно. Как хороший архитектурный проект: чистые линии, никакого лишнего.

- Тон: экспертный, прямой, без канцелярита
- Декора минимум: тонкие бордеры, лёгкие тени, одна акцентная линия
- Сетка: строгая, 12-колонок, с чёткими отступами
- Никаких stock-фото бизнесменов, никаких иконок-эмодзи

## 5. Обязательные элементы

### Кнопки

**Основная (CTA):**
```css
background: #F97316;
color: #0D0F11;
border-radius: 8px;
padding: 14px 32px;
font-weight: 700;
font-size: 16px;
transition: background 0.2s ease, transform 0.15s ease;
```
Hover: `background: #FB923C`, слабый `transform: translateY(-1px)`.

**Вторичная (ghost):**
```css
background: transparent;
border: 1px solid rgba(255,255,255,0.2);
color: #F1F5F9;
```
Hover: `border-color: #F97316`, `color: #F97316`.

### Формы

Форма записи на разбор:
- Поля: Имя (text), Мессенджер/email (text), необязательно — «Расскажите в 1-2 строках о компании» (textarea)
- Кнопка отправки
- После отправки: inline-сообщение «Получила. Напишу в течение дня.» (без перезагрузки страницы)
- Валидация: только имя и контакт обязательны, ошибка — красная рамка + текст под полем

### Анимации

**Scroll-triggered reveal:**
- Секции и карточки появляются при скролле: `opacity: 0` → `opacity: 1`, `translateY(20px)` → `translateY(0)`
- Задержка между карточками в ряду: 0.1s stagger
- Реализация: Intersection Observer API, без библиотек

**Счётчики (секция PROOF):**
- Цифры считаются от 0 до значения при первом попадании в viewport
- Длительность: 1500ms, easing: ease-out

**Навигация:**
- Sticky header с blur-фоном: `backdrop-filter: blur(12px); background: rgba(13,15,17,0.85);`
- При скролле вниз — хедер скрывается, при скролле вверх — появляется

**Схема K-A-Q-S (Hero):**
- 4 буквы-блока появляются последовательно с задержкой
- Линии между блоками анимируются через `stroke-dashoffset` (SVG)

**Hover-эффекты:**
- Карточки: лёгкий подъём `translateY(-4px)` + усиление бордера до `rgba(249,115,22,0.3)`
- Модули K-A-Q-S: при hover — акцентный цвет на букве + небольшое свечение

### Навигация (хедер)

Логотип `K-A-Q-S™` слева + пункты меню:
`Методология` / `Кейсы` / `О себе` / `Форматы` — и кнопка `Пройти диагностику` справа (оранжевая).

На мобайле: бургер → боковое меню (drawer) справа, оверлей.

### Мобильная версия

- Контентные колонки → одна колонка
- Карточки модулей → вертикальный стек (scroll)
- Кнопки → full-width
- Шрифты: H1 → 36–40px на мобайле
- Минимальная ширина: 320px

## 6. Технические требования

- Один самодостаточный HTML-файл: разметка + CSS + JS внутри
- Никаких внешних библиотек (Bootstrap, jQuery, Alpine.js и т.д.)
- Шрифты: Google Fonts через `<link>`
- Адаптивность: от 320px, flexbox / grid, без фиксированных пиксельных ширин
- `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- Форма: обработка через fetch (без перезагрузки), можно заглушку `console.log` до подключения бэкенда
- Анимации: только через CSS Transitions + JS Intersection Observer

## 7. Рекомендации экспертов из исследования

### Конкурентный анализ → позиционирование

Все крупные аналоги (IBM, Gartner, Cisco, Microsoft) имеют один провал: результат — это реклама их продуктов, нет независимого роадмапа. **На сайте явно показать, что K-A-Q-S™ — независимая методология, не ведущая к покупке чужого ПО.** Это ключевое конкурентное преимущество.

Gartner закрыт за paywall $30k+/год, IBM — только Enterprise. **Использовать на сайте формулировку: «для компаний 20–500 человек, которых игнорирует Gartner».** Это прямое позиционирование против конкурентов без их упоминания.

### UX-эксперт → поведение на сайте

Топ-рекомендации из исследования, реализовать обязательно:

1. **Прогресс-бар с шагами** — на самой платформе диагностики (не на лендинге, но упомянуть в описании)
2. **Социальное доказательство с числами** — счётчик компаний в секции PROOF, конкретные цифры результатов в кейсах
3. **Конкретный следующий шаг после результатов** — не «ваш уровень 3», а «сделайте X, Y, Z в ближайшие 90 дней». Транслировать это на лендинге через описание продукта.
4. **Мобильная версия как приоритет**, не afterthought
5. **PDF-отчёт как артефакт** — явно показать пример страницы PDF в секции HOW IT WORKS

### Монетизация → воронка лендинга

Рекомендуемая стратегия на старте: **Freemium → лидогенерация консультаций**. Лендинг должен это отражать:

- Первый CTA — бесплатная диагностика (снизить барьер)
- Второй CTA — записаться на разбор (высокий intent)
- Конверсия в консультацию: 10–20% от прошедших диагностику (данные из исследования)
- Цена консультации: 50–150k ₽ — не указывать на лендинге, обсуждать лично

**Не показывать тарифы на лендинге** на первом этапе. Только два действия: диагностика или разбор.
