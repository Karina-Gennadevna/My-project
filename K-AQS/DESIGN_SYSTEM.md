# K-A-Q-S™ Design System
_Версия 1.0 — 2026-03-06_

---

## Визуальная концепция

**Referents:** Palantir Foundry × McKinsey.com × Apple Developer × Bloomberg Terminal

Платформа выглядит как инструмент для серьёзных людей, которые принимают решения на основе данных. Не стартап, не агентство, не SaaS с конфетти. Дорого, сдержанно, интеллектуально.

**Три визуальных принципа:**
1. **Signal over noise** — каждый элемент несёт смысл. Декор только если он усиливает структуру.
2. **Dark precision** — тёмный фон не потому что «модно», а потому что данные читаются лучше на тёмном. Как Bloomberg, как Palantir.
3. **Controlled tension** — оранжевый акцент применяется редко и точно. Он должен обозначать действие или важность, не украшать.

## 1. Цветовая система

### Фоны (Background)

| Токен | HEX | Применение |
|-------|-----|------------|
| `bg-base` | `#080A0C` | Основной фон страницы |
| `bg-surface` | `#0F1215` | Карточки, панели, секции поверх base |
| `bg-elevated` | `#161B20` | Всплывающие элементы, модальные окна, tooltip |
| `bg-overlay` | `#1C2229` | Hover-состояния карточек, аккордеоны при открытии |
| `bg-subtle` | `#0C0E11` | Чередующиеся секции, alt-фон |

**Правило:** Глубина создаётся не через яркость, а через ступени тёмного. Никаких светлых секций. Единственный способ разбить монотонность — тонкие бордеры и разные уровни surface.

---

### Текст (Typography Colors)

| Токен | HEX | Применение |
|-------|-----|------------|
| `text-primary` | `#F0F4F8` | Заголовки, основной контент |
| `text-secondary` | `#8A97A8` | Подзаголовки, описания, вспомогательный текст |
| `text-muted` | `#4A5568` | Метаданные, подписи, placeholder |
| `text-accent` | `#F97316` | Акцентные слова, выделения, активные состояния |
| `text-inverse` | `#080A0C` | Текст на оранжевых кнопках |

**Правило:** Никогда не использовать чистый белый `#FFFFFF`. `text-primary` — это тёплый холодный белый с лёгким голубоватым оттенком. Это снижает усталость глаз и делает интерфейс «аналитическим», а не «рекламным».

---

### Акцентный цвет (Accent)

| Токен | HEX | Применение |
|-------|-----|------------|
| `accent-primary` | `#F97316` | CTA-кнопки, активные состояния, ключевые элементы |
| `accent-hover` | `#EA6C0A` | Hover на оранжевых элементах |
| `accent-muted` | `#7C3B0F` | Фон тегов, бейджей, подсветка строк |
| `accent-glow` | `rgba(249,115,22,0.12)` | Box-shadow, свечение вокруг активных элементов |
| `accent-line` | `rgba(249,115,22,0.4)` | Декоративные линии, активная вкладка |

**Правило:** Оранжевый занимает не более 5–8% визуального пространства экрана. Если больше — он теряет вес. Это цвет действия и внимания, не цвет декора.

---

### Линии и бордеры (Borders)

| Токен | HEX / значение | Применение |
|-------|----------------|------------|
| `border-subtle` | `rgba(255,255,255,0.05)` | Разделители внутри секций, между карточками |
| `border-default` | `rgba(255,255,255,0.08)` | Бордер карточек, инпутов, панелей |
| `border-emphasis` | `rgba(255,255,255,0.14)` | Бордер при hover, активные карточки |
| `border-accent` | `rgba(249,115,22,0.35)` | Акцентный бордер: выбранный тариф, активный блок |

**Правило:** Бордеры всегда полупрозрачные. Никаких сплошных серых линий — это выглядит дёшево. Полупрозрачность создаёт ощущение слоёв и глубины.

---

### Свечения и тени (Glow & Shadow)

| Назначение | Значение |
|-----------|---------|
| Карточка, default | `0 1px 3px rgba(0,0,0,0.4)` |
| Карточка, hover | `0 4px 24px rgba(0,0,0,0.6), 0 0 0 1px rgba(255,255,255,0.08)` |
| CTA-кнопка | `0 0 20px rgba(249,115,22,0.25)` |
| CTA-кнопка, hover | `0 0 32px rgba(249,115,22,0.4)` |
| Активный блок K/A/Q/S | `0 0 40px rgba(249,115,22,0.08)` |
| Модальное окно | `0 24px 64px rgba(0,0,0,0.8)` |

**Правило:** Glow — это не эффект «красиво». Это иерархия. Светится то, что важно и активно. Фоновые элементы не светятся.

---

### Вспомогательные цвета (Semantic)

| Назначение | HEX | Применение |
|-----------|-----|------------|
| Успех / позитивный сигнал | `#22C55E` (осторожно) | Только в диаграммах, только как малый индикатор |
| Предупреждение / риск | `#EAB308` | Метка среднего риска в отчётах |
| Критический риск | `#EF4444` | Критические показатели в диагностике |
| Нейтральный | `#64748B` | Пустые состояния, отключённые элементы |

**Правило:** Эти цвета — только в контексте данных (диаграммы, метки, шкалы). На лендинге — максимально ограниченно. На лендинге смысловая нагрузка несётся оранжевым и белым.

## 2. Типографика

### Шрифтовой стек

**Основной шрифт:** `Inter` (Google Fonts)
- Причина: нейтральный, современный, идеален для UI и длинного текста на тёмном фоне. Используется Bloomberg, Linear, Vercel.

**Моноширинный шрифт:** `IBM Plex Mono` (Google Fonts)
- Применение: числа в диаграммах, коды, технические метки, цифры метрик
- Причина: придаёт «аналитический» и «терминальный» характер. Ссылка на Palantir/Bloomberg.

**Нет:** decorative fonts, serif для UI, display-шрифты. Только Inter + IBM Plex Mono.

---

### Масштаб заголовков

| Токен | Размер | Weight | Line-height | Letter-spacing | Применение |
|-------|--------|--------|-------------|----------------|------------|
| `display-xl` | 72px / 4.5rem | 700 | 1.05 | -0.03em | Hero-заголовок (экран 1) |
| `display-lg` | 56px / 3.5rem | 700 | 1.08 | -0.025em | Заголовки крупных секций |
| `h1` | 44px / 2.75rem | 600 | 1.1 | -0.02em | Заголовки продуктовых экранов |
| `h2` | 32px / 2rem | 600 | 1.2 | -0.015em | Подзаголовки секций |
| `h3` | 24px / 1.5rem | 600 | 1.3 | -0.01em | Заголовки карточек, блоков |
| `h4` | 18px / 1.125rem | 600 | 1.4 | 0 | Подписи блоков, тарифы |

**Правило:** Заголовки сжатые (tight line-height, negative letter-spacing). Это создаёт ощущение плотности и веса. Так делает Palantir, Apple, Linear. Рыхлые заголовки выглядят как шаблон.

---

### Масштаб текста (Body)

| Токен | Размер | Weight | Line-height | Применение |
|-------|--------|--------|-------------|------------|
| `body-lg` | 18px / 1.125rem | 400 | 1.7 | Вводные абзацы, подзаголовки секций |
| `body-md` | 16px / 1rem | 400 | 1.65 | Основной текст, описания |
| `body-sm` | 14px / 0.875rem | 400 | 1.6 | Вторичный текст, подписи карточек |
| `caption` | 12px / 0.75rem | 400 | 1.5 | Метаданные, timestamp, footnote |
| `label` | 12px / 0.75rem | 600 | 1 | uppercase, letter-spacing 0.1em — бейджи, теги, навигация |
| `mono-lg` | 20px / 1.25rem | 500 | 1 | Числа метрик, показатели |
| `mono-sm` | 13px / 0.8125rem | 400 | 1.4 | Технические подписи, коды |

---

### Типографические правила

1. **Заголовки** — `text-primary`, никогда не `text-accent` для длинных заголовков
2. **Акцент в заголовке** — отдельное слово или фраза может быть `text-accent`, не весь заголовок
3. **Body text** — `text-secondary` для описаний, `text-primary` только для ключевых утверждений
4. **Uppercase** — только `label` (12px, weight 600, letter-spacing 0.1em). Нигде больше.
5. **Жирный текст внутри абзаца** — weight 600, `text-primary`, для выделения тезиса
6. **Никогда:** italic для UI-текста, underline без hover, text-decoration в навигации

## 3. Сетка и Layout

### Контейнер

| Размер | Ширина контейнера | Применение |
|--------|-------------------|------------|
| Узкий | max-width: 720px | Текстовые блоки, заголовки секций, FAQ |
| Стандартный | max-width: 1100px | Основной контент, карточки |
| Широкий | max-width: 1280px | Диаграммы, тарифы, полноширинные секции |
| Полный | 100% | Фоны секций, Hero, CTA |

**Боковые поля (padding):** 24px (mobile) / 40px (tablet) / 80px (desktop)

---

### Колоночная сетка

| Контекст | Колонки | Gap |
|---------|---------|-----|
| Desktop | 12 колонок | 24px |
| Tablet (768px) | 8 колонок | 20px |
| Mobile (375px) | 4 колонки | 16px |

**Типовые раскладки:**

- **Hero:** 1 колонка, центрирование, max-width 800px для текста
- **Карточки K-A-Q-S:** 4 колонки (desktop) → 2 (tablet) → 1 (mobile)
- **Продуктовые экраны:** 2 колонки (текст + мокап) → 1 (mobile)
- **Тарифы:** 3 колонки (desktop) → 1 (mobile, с горизонтальным скроллом или стек)
- **Проблемы/симптомы:** 2 колонки (desktop) → 1 (mobile)

---

### Вертикальный ритм секций

| Тип | Padding top/bottom |
|-----|-------------------|
| Hero | 120px / 100px |
| Основные секции | 96px |
| Компактные секции | 64px |
| CTA финальный | 120px |
| Разделители между блоками внутри секции | 48px |

**Правило:** Воздух — это роскошь. Dark enterprise не экономит на пустом пространстве. Большие отступы сигнализируют: «здесь нет мусора, только суть».

---

### Разделители секций

Три варианта разделения секций, не смешивать:
1. **Тонкая горизонтальная линия** `border-subtle` — между плотно связанными блоками
2. **Смена background** — `bg-base` ↔ `bg-surface` ↔ `bg-subtle` — между тематическими секциями
3. **Пустое пространство** — просто большой padding — когда контент говорит сам за себя

Никаких волн, скошенных краёв, clip-path-сепараторов. Только прямые линии.

## 4. UI-компоненты

### Кнопки

**Primary CTA (оранжевая)**
- Background: `accent-primary` (#F97316)
- Text: `text-inverse` (#080A0C), weight 600, 15px
- Padding: 14px 28px
- Border-radius: 6px
- Box-shadow: `0 0 20px rgba(249,115,22,0.25)`
- Hover: background → `accent-hover`, shadow → glow усиливается, transform: translateY(-1px)
- Transition: все свойства 200ms ease-out
- Cursor: pointer
- Никаких uppercase, никаких стрелок внутри кнопки (стрелка — только как отдельный inline-элемент)

**Secondary (контурная)**
- Background: transparent
- Border: 1px solid `border-default`
- Text: `text-primary`, weight 500, 15px
- Padding: 14px 28px
- Border-radius: 6px
- Hover: border → `border-emphasis`, background → `bg-overlay`, слабый glow

**Ghost (текстовая)**
- Background: transparent, no border
- Text: `text-secondary`, weight 500, 14px
- Hover: text → `text-primary`
- Применение: «Узнать как работает →», навигационные ссылки

**Правило размеров:**
- Large (Hero, CTA-секции): padding 16px 32px, font-size 16px
- Medium (продуктовые экраны): padding 14px 28px, font-size 15px
- Small (карточки, внутри блоков): padding 10px 20px, font-size 13px

---

### Карточки

**Базовая карточка**
- Background: `bg-surface`
- Border: 1px solid `border-default`
- Border-radius: 10px
- Padding: 28px 32px
- Box-shadow: default card shadow
- Hover: border → `border-emphasis`, shadow → hover card shadow, background → `bg-overlay`
- Transition: 250ms ease-out

**Карточка K-A-Q-S (с буквой)**
- Та же база
- Сверху: большая буква (K/A/Q/S), color `accent-primary`, font-size 48px, font IBM Plex Mono, opacity 0.9
- Тонкая верхняя полоска цвета акцента (2px, border-top: `accent-line`)
- На hover: слабый accent-glow вокруг всей карточки

**Карточка тарифа**
- Та же база + фиксированная высота
- «Рекомендуем» тариф: border → `border-accent`, background → чуть темнее с оранжевым оттенком (`rgba(249,115,22,0.04)`)
- Бейдж «Рекомендуем» — абсолютно позиционированный, вверху карточки

---

### Бейджи и теги

**Тег / категория**
- Background: `rgba(249,115,22,0.1)` (accent-muted)
- Border: 1px solid `rgba(249,115,22,0.2)`
- Text: `text-accent`, font-size 12px, weight 600, uppercase, letter-spacing 0.08em
- Padding: 4px 10px
- Border-radius: 4px
- Применение: над заголовками секций («Диагностика», «Платформа», «01 / Шаг первый»)

**Нейтральный тег**
- Background: `bg-elevated`
- Border: 1px solid `border-subtle`
- Text: `text-secondary`, font-size 12px, weight 500
- Применение: технические метки, статусы, метаданные

**Правило:** Бейджи — это иерархия. Не использовать их для украшения. Только для категоризации или важных меток.

---

### Блоки с данными (Data blocks)

Применение: симптомы на экране 2, тезисы на экране 3, принципы на экране 10.

**Структура:**
- Иконка (SVG, 20×20px, color `text-muted` или `accent-primary`)
- Заголовок блока: `h4`, `text-primary`
- Описание: `body-sm`, `text-secondary`
- Нет бордера. Разделение через gap сетки.

**Альтернатива с числом:**
- Большое число или буква вместо иконки (IBM Plex Mono, 40–48px, `text-muted` или `accent-primary` с opacity 0.5)
- Используется для шагов, индексов, измерений

---

### Модальное окно (если нужно)

- Background: `bg-elevated`
- Border: 1px solid `border-default`
- Border-radius: 12px
- Padding: 40px 40px 32px
- Box-shadow: modal shadow
- Backdrop: `rgba(0,0,0,0.7)`, blur(4px)
- Анимация: fade-in 200ms + scale from 0.97 to 1
- Закрытие: Esc, клик на backdrop, крестик (24px, `text-muted`, hover → `text-primary`)
- Ширина: max-width 520px (стандартное) / 760px (широкое)

---

### FAQ / Аккордеон

- Каждый вопрос — отдельный блок
- Separator: `border-subtle` между вопросами
- Состояние закрыто: заголовок `body-md` weight 500, `text-primary` + иконка «+» справа `text-muted`
- Состояние открыто: иконка «×» или «−», фон вопроса → `bg-overlay`, текст ответа появляется снизу
- Ответ: `body-sm`, `text-secondary`, padding-top 12px, padding-bottom 20px
- Анимация: высота раскрывается за 220ms ease-out (max-height transition)
- Нет кастомных стрелок с тяжёлой анимацией — только простой плюс/минус

---

### Навигация

- Background: `rgba(8,10,12,0.85)`, backdrop-filter: blur(16px) saturate(1.5)
- Тонкая нижняя линия: `border-subtle`
- Высота: 64px
- Логотип: слева, `text-primary`, font-weight 700, letter-spacing -0.02em
- Символ ™ — font-size 0.6em, vertical-align super, `text-muted`
- Ссылки навигации: `text-secondary`, font-size 14px, weight 500, hover → `text-primary`, transition 150ms
- CTA в навигации: Secondary button small
- Поведение при скролле: навигация фиксируется сверху, border-bottom чуть усиливается

---

### Разделительная линия с текстом (декоративная)

Применение: между крупными секциями, как контекстный маркер.
```
———  K-A-Q-S™  ———
```
- Размер: 11px, uppercase, `text-muted`, letter-spacing 0.15em
- Линии: тонкие горизонтальные, `border-subtle`
- Используется максимум 2–3 раза на всей странице

## 5. Motion — анимации и взаимодействие

### Принцип

**Движение должно объяснять, не развлекать.**

Любая анимация на K-A-Q-S™ работает по правилу: если убрать анимацию, смысл сохраняется. Анимация только усиливает восприятие иерархии и переходов.

---

### Hover-состояния

| Элемент | Поведение |
|---------|-----------|
| CTA-кнопка primary | background темнеет, glow усиливается, translateY(-1px) |
| Secondary кнопка | border светлеет, background чуть поднимается |
| Карточка | border светлеет, тень глубже, translateY(-2px) |
| Навигационная ссылка | color → text-primary, 150ms |
| FAQ строка | фон → bg-overlay, 200ms |
| Буква K/A/Q/S в карточке | opacity 1 (было 0.8), glow появляется, 200ms |
| Тег/бейдж | border → border-emphasis |

**Все hover-переходы:** ease-out, 150–250ms. Никогда не bounce, не elastic, не spring.

---

### Reveal on scroll (появление при скролле)

**Принцип:** Элементы появляются снизу вверх с небольшим смещением. Нет мгновенного pop-in, нет тяжёлого zoom.

| Элемент | Начальное состояние | Финальное | Длительность | Delay |
|---------|--------------------|-----------|----|-------|
| Заголовок секции | opacity: 0, translateY(20px) | opacity: 1, translateY(0) | 500ms | 0ms |
| Подзаголовок | opacity: 0, translateY(16px) | opacity: 1, translateY(0) | 500ms | 80ms |
| Карточки (каждая) | opacity: 0, translateY(24px) | opacity: 1, translateY(0) | 450ms | 0 + 80ms × index |
| Body text-блоки | opacity: 0 | opacity: 1 | 400ms | 100ms |

**Реализация:** Intersection Observer (порог 0.15). CSS-классы `.reveal` → `.visible`. Без JS-анимационных библиотек.

**Максимальный stagger:** не более 4 элементов с delay. Если карточек 4 — delay 0 / 80 / 160 / 240ms. Если карточек 8 — первые 4 со stagger, остальные одновременно.

---

### Микро-взаимодействия

| Взаимодействие | Поведение |
|---------------|-----------|
| Клик на кнопку | scale: 0.97 за 100ms, возврат за 200ms |
| FAQ открытие | height анимируется от 0 до auto, 220ms ease-out |
| Переход между вкладками (если есть) | контент fade 200ms, active indicator скользит |
| Копирование email (футер) | tooltip «Скопировано» fade-in/out, 1.5s |
| Фокус на инпут (форма) | border → `border-accent`, subtle glow |
| Checkbox/toggle (если есть) | плавный slide, 200ms |

---

### Диаграммы K-A-Q-S (на экране результатов / мокап на лендинге)

**Radar chart (паутина):**
- Анимация: линии и заливка рисуются от центра за 800ms ease-out
- Stroke: `accent-primary` с opacity 0.8
- Fill: `rgba(249,115,22,0.08)`
- Сетка: `border-subtle`
- Метки: IBM Plex Mono, 11px, `text-muted`

**Шкалы (bar/progress):**
- Анимация: ширина от 0% до значения за 600ms ease-out
- Задержка: каждая шкала — +100ms stagger
- Fill: `accent-primary`
- Track: `bg-elevated`
- Высота: 4px (тонко, аналитично)

**Числа (counter):**
- Анимация: count up от 0 до значения за 1000ms ease-out
- Шрифт: IBM Plex Mono
- Применение: метрики в hero или на экране «о платформе»

**Правило:** Диаграммы анимируются только один раз — при первом появлении в viewport. Повторная анимация при скролле назад — нет.

## 6. Визуальные детали и текстуры

### Фоновые элементы (subtle)

Допустимые фоновые детали — только CSS, без изображений:

1. **Точечная сетка (dot grid)**
   - `background-image: radial-gradient(circle, rgba(255,255,255,0.04) 1px, transparent 1px)`
   - `background-size: 24px 24px`
   - Применение: Hero, CTA — только там, не везде

2. **Тонкие линии (grid lines)**
   - Горизонтальные или вертикальные полупрозрачные линии
   - `rgba(255,255,255,0.03)`, не более 1px
   - Применение: фон экрана диагностики, технические секции

3. **Радиальный градиент-подсветка**
   - `radial-gradient(ellipse 60% 40% at 50% 0%, rgba(249,115,22,0.06), transparent)`
   - Применение: Hero — очень слабо, за заголовком. Только один раз на странице.

**Запрещено:** шум/noise texture, animated particles, SVG-орнаменты, glassmorphism (blur поверх контента), неоновые свечения.

---

### Иконки

- **Стиль:** outline, 1.5px stroke, с закруглёнными концами (stroke-linecap: round)
- **Размеры:** 16px (inline), 20px (карточки, списки), 24px (навигация, кнопки)
- **Цвет:** `text-muted` по умолчанию, `text-accent` для активных/важных
- **Источник:** Lucide Icons (SVG-спрайт или inline SVG). Никаких emoji, никаких Font Awesome.
- **Правило:** Иконка всегда сопровождается текстом. Иконка без текста — только если смысл абсолютно очевиден (закрыть, поиск, стрелка).

---

### Декоративные элементы (единственные допустимые)

1. **Номера шагов:** `01 / 02 / 03` — IBM Plex Mono, 48–64px, `text-muted` opacity 0.3, за заголовком шага
2. **Буквы K-A-Q-S:** крупные одиночные буквы, opacity 0.06–0.1, на фоне соответствующих блоков
3. **Горизонтальная оранжевая черта:** 2px, width 40–48px, над заголовком секции — как якорный элемент

## 7. Отзывчивость (Breakpoints)

| Breakpoint | Значение | Поведение |
|-----------|---------|-----------|
| Mobile | 320px–767px | 1 колонка, все сетки схлопываются, display-xl → 40px |
| Tablet | 768px–1023px | 2 колонки, сетки K-A-Q-S → 2×2 |
| Desktop | 1024px–1279px | Полная сетка, стандартные размеры |
| Wide | 1280px+ | max-width ограничивает контент, фон тянется |

**Типографика на мобиле:**
- `display-xl` (72px) → 36px
- `display-lg` (56px) → 30px
- `h1` (44px) → 26px
- `h2` (32px) → 22px
- `body-lg` (18px) → 16px

**Кнопки на мобиле:** width: 100% для основных CTA. Стекуются вертикально.

**Навигация на мобиле:** hamburger-меню. Overlay на `bg-elevated` с анимацией slide-in справа.

## 8. Чеклист перед реализацией

- [ ] Только Inter + IBM Plex Mono — никаких других шрифтов
- [ ] Оранжевый — не более 8% площади любого экрана
- [ ] Все бордеры полупрозрачные, никаких сплошных серых
- [ ] Hover на каждом интерактивном элементе
- [ ] Reveal-анимация только через IntersectionObserver
- [ ] Никаких JS-анимационных библиотек (GSAP, AOS, Framer)
- [ ] Диаграммы чистые SVG или CSS
- [ ] Навигация sticky с blur-backdrop
- [ ] Все секции корректны от 320px
- [ ] Нет изображений в дизайне (только мокапы UI как CSS-элементы)
- [ ] Нет stock-фотографий, нет иллюстраций, нет декоративных blob/wave
- [ ] Только прямые края, никаких clip-path-сепараторов между секциями
