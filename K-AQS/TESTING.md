## 1. Архитектура проекта

Проект состоит из **трёх независимых частей**. Каждая тестируется отдельно.

| Часть | Что это | Как запускать |
|---|---|---|
| **A. Next.js лендинг** | `app/`, `components/` — React 19 + Next.js 16 | `npm run dev` |
| **B. HTML-диагностика** | 6 статических `.html` файлов в корне | открыть в браузере |
| **C. Telegram Mini App** | `tg-app/index.html` — self-contained | открыть в браузере |

Части **не связаны между собой** технически. Навигация между ними — только через HTML-ссылки.

## 2. Запуск проекта

### Часть A — Next.js лендинг

**Требования:** Node.js ≥ 18, npm ≥ 9

```bash
cd K-AQS
npm install
npm run dev
```

Открыть: `http://localhost:3000`

**Дополнительные команды:**
```bash
npm run build   # production сборка
npm run lint    # ESLint (встроенный Next.js)
npm start       # запуск production сборки (после build)
```

**Ожидаемый результат:** лендинг открывается без ошибок в консоли.

---

### Часть B — HTML-диагностика

Файлы открываются напрямую в браузере (без сервера):

| Файл | URL / путь |
|---|---|
| `kaqs-landing.html` | Лендинг платформы |
| `kaqs-auth.html` | Форма входа / регистрации |
| `kaqs-wizard.html` | Викинг-диагностика (32 вопроса) |
| `kaqs-results.html` | Результаты диагностики |
| `kaqs-dashboard.html` | Личный кабинет |
| `kaqs-preview.html` | Preview-страница |

> Все файлы self-contained. Внешних зависимостей нет, интернет не требуется (кроме Google Fonts).

### Часть C — Telegram Mini App

Файл: `tg-app/index.html`

**Вариант 1 — браузер (без Telegram):**
Открыть `tg-app/index.html` напрямую в браузере. Telegram SDK заменяется mock-объектом — приложение работает полностью.

**Вариант 2 — в Telegram:**
Mini App деплоится на `https://mini-app-sandy-eight.vercel.app/` (константа `APP_URL` в коде, строка ~1435 оригинального файла). Для тестирования внутри Telegram нужен доступ к боту.

**Вариант 3 — локальный сервер:**
```bash
cd tg-app
npx serve .
# или
python -m http.server 8080
```
Затем открыть `http://localhost:8080`.

## 4. Граничные случаи

### Mini App — крайние значения индекса

| Индекс | Ожидаемый уровень | Цвет | Риск |
|---|---|---|---|
| 0–39 | Критический | `#EF4444` | Критический |
| 40–59 | Нестабильный | `#F97316` | Высокий |
| 60–74 | Средний | `#FBBF24` | Умеренный |
| 75–89 | Хороший | `#22C55E` | Низкий |
| 90–100 | Высокий | `#10B981` | Низкий |

**Как получить конкретный индекс:**
- Все ответы «Да — работает стабильно» (score=4) → индекс ~100
- Все ответы «Нет» (score=1) → индекс ~0
- Чередование → промежуточные значения

Расчёт индекса: среднее по 5 зонам. Каждая зона = среднее по 3 основным вопросам (макс. 12 баллов → нормируется до 100) ± корректировка контрольным вопросом (±4 или ±8 баллов).

### Mini App — localStorage

- История хранится в `localStorage.kaqs_history` как JSON-массив `[{date, index}, ...]`
- Максимум 30 записей (старые удаляются)
- При очистке хранилища (DevTools → Application → Storage) — граф и история пустые

### Wizard — граничные случаи

- Нажать «Далее» без ответов → кнопка должна быть disabled
- Браузер без JavaScript → ничего не работает (JS-зависимые страницы)
