**Цель:** получить живую ссылку, которую можно отправить в Telegram
**Время:** 20 минут
**Результат:** https://kaqs.netlify.app — демо доступно всем

Что делаем:
1. Идём на [netlify.com](https://netlify.com) → Sign up (бесплатно)
2. На главной странице — перетащить папку `K-AQS/` прямо в браузер
3. Netlify генерирует URL автоматически
4. (опционально) переименовать в настройках: kaqs-platform → красивый URL

> **Для тебя:** Netlify — это хостинг. Ты просто перетаскиваешь папку как файл в почту,
> а Netlify превращает её в сайт с адресом в интернете. Бесплатно. Без кода.

Проверка: открываем ссылку на телефоне — сайт работает.

## Шаг 2 — Установить Node.js
**Цель:** открыть доступ к настоящему бэкенду (авторизация, база данных)
**Время:** 10 минут (скачать + установить)
**Результат:** команда `node --version` показывает версию

Что делаем:
1. Идём на [nodejs.org](https://nodejs.org) → скачиваем **LTS** версию (левая кнопка)
2. Запускаем установщик, везде жмём Next
3. Перезапускаем VS Code
4. В терминале проверяем: `node --version`

> **Для тебя:** Node.js — это "движок", который позволяет запускать JavaScript не в браузере,
> а на компьютере. Без него Next.js (наш бэкенд) не запустится.
> Это как Word для .docx файлов — без него файл не откроется.

Проверка: `node --version` выдаёт что-то вроде `v22.x.x`.

## Шаг 3 — Запустить бэкенд локально
**Цель:** запустить настоящий Next.js проект на localhost
**Время:** 30 минут
**Результат:** http://localhost:3000 — живая платформа с базой данных

Что делаем:
1. В терминале: `cd "e:/Вайбкодинг/K-AQS/kaqs-platform"`
2. Устанавливаем зависимости: `npm install`
3. Создаём проект в [Supabase](https://supabase.com) (бесплатно) → копируем DATABASE_URL
4. Обновляем файл `.env.local` — вставляем DATABASE_URL и генерируем JWT_SECRET
5. Создаём таблицы в базе: `npx prisma db push`
6. Запускаем: `npm run dev`
7. Открываем http://localhost:3000

> **Для тебя:** npm install — это как "установить все приложения по списку из пакета".
> Supabase — облачная база данных. Там будут храниться пользователи и их оценки.
> npm run dev — запуск нашего сайта на твоём компьютере.

Проверка: регистрируемся → проходим опросник → видим результаты → PDF.

## Шаг 4 — Деплой бэкенда на Vercel
**Цель:** перенести настоящую платформу из localhost в интернет
**Время:** 20 минут
**Результат:** https://kaqs-platform.vercel.app — реальная рабочая платформа

Что делаем:
1. Создаём репозиторий на [github.com](https://github.com)
2. В терминале: `git init && git add . && git commit -m "init"`
3. `git remote add origin <url> && git push`
4. Идём на [vercel.com](https://vercel.com) → Import Git Repository
5. Выбираем репозиторий → Vercel сам находит Next.js
6. Добавляем Environment Variables (из .env.local)
7. Deploy → получаем URL

> **Для тебя:** GitHub — это хранилище кода в интернете (как Google Drive, но для программистов).
> Vercel — хостинг, который умеет запускать Next.js и следит за обновлениями в GitHub.
> Как только ты сохраняешь изменения в GitHub — Vercel сам обновляет сайт.

Проверка: регистрируемся на живом URL, проходим оценку, скачиваем PDF.

## Шаг 5 — Подключить оплату
**Цель:** принимать деньги за платный PDF-отчёт
**Время:** 2–3 часа (регистрация в ЮKassa требует документы)
**Результат:** кнопка "Получить полный отчёт — 4 900 ₽" реально работает

Что делаем:
1. Регистрируемся в [ЮKassa](https://yookassa.ru) как ИП/самозанятый
2. Получаем shopId и secretKey
3. Добавляем в .env + создаём API-маршрут `/api/payment`
4. Добавляем на страницу результатов кнопку оплаты
5. После оплаты — разблокируем полный отчёт + PDF

> **Для тебя:** ЮKassa — это платёжный шлюз. Как касса в магазине.
> Пользователь нажимает кнопку → попадает на страницу оплаты ЮKassa →
> вводит карту → деньги приходят тебе.

Проверка: тестовый платёж проходит, PDF разблокируется.
