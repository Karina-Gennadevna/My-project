# Процесс: Контент-маркетинг и ведение соцсетей

**Бизнес:** Платформа управления бизнесом / Диагностика индекса управляемости
**Каналы:** Telegram-канал, ВКонтакте (планируется)
**Дата:** 2026-03-19

---

## 1. ЧТО ЗАПУСКАЕТ ПРОЦЕСС?

### Основной триггер — плановый (регулярный)
- Наступает понедельник → нужно опубликовать контент на текущую неделю
- Наступает 1-е число → нужно спланировать контент на следующий месяц

### Дополнительные триггеры — ситуативные
| Триггер | Пример |
|---------|--------|
| Событие в продукте | Вышла новая функция платформы |
| Активность аудитории | Пришёл частый вопрос от клиентов |
| Инфоповод | Кейс клиента, результат диагностики |
| Провал метрик | Просмотры упали → нужен взрывной пост |

### Что нужно на входе
- Тема или рубрика поста
- Целевая аудитория (владельцы бизнеса)
- Ключевое сообщение / боль / польза
- Канал публикации (Telegram / ВК / оба)
- Дата и время публикации

## 2. ПОШАГОВЫЕ ДЕЙСТВИЯ

### Фаза A — Планирование (1 раз в месяц, ~3–4 часа)

| # | Шаг | Кто | Что делает | Время | Данные |
|---|-----|-----|-----------|-------|--------|
| A1 | Определить темы | Основатель | Выбрать 8–12 тем на месяц по рубрикам | 30 мин | Боли ЦА, вопросы клиентов, продуктовый роадмап |
| A2 | Заполнить контент-план | Основатель | Внести в таблицу: дата, тема, рубрика, канал, формат | 20 мин | Шаблон Google Sheets |
| A3 | Написать бриф на каждый пост | Основатель | 2–3 строки: о чём пост, боль, главная мысль, CTA | 60 мин | Контент-план |
| A4 | Согласовать план | Основатель | Просмотреть, скорректировать | 15 мин | Готовый план |

**Итого фаза A:** ~2–2.5 часа в месяц

### Фаза B — Создание контента (еженедельно, каждый пост)

| # | Шаг | Кто | Что делает | Время | Данные |
|---|-----|-----|-----------|-------|--------|
| B1 | Взять бриф из плана | Основатель | Открыть таблицу, найти пост на сегодня | 2 мин | Google Sheets |
| B2 | Написать черновик | Основатель | Текст поста 500–1500 символов | 30–60 мин | Бриф, голос бренда |
| B3 | Отредактировать | Основатель | Убрать лишнее, проверить структуру | 15–20 мин | Черновик |
| B4 | Сделать визуал | Основатель / дизайнер | Картинка, обложка, схема | 20–40 мин | Текст, шаблон Canva |
| B5 | Адаптировать для ВК | Основатель | Переписать под формат ВК (другая подача) | 15–20 мин | Telegram-пост |
| B6 | Запланировать публикацию | Основатель | Поставить в SMMPlanner / Combot / вручную | 5–10 мин | Готовый пост, дата |

**Итого на один пост:** 1.5–2.5 часа
**Итого в неделю (2–3 поста):** 3–7 часов

## 3. ЧТО НА ВЫХОДЕ?

### Обязательный результат каждого цикла:
- ✅ 2–3 поста в неделю опубликованы в Telegram-канале
- ✅ Те же посты (адаптированные) опубликованы в ВК
- ✅ Статистика занесена в таблицу
- ✅ Следующие посты запланированы заранее (не в последний момент)

### Метрики успешности:
| Метрика | Цель |
|---------|------|
| Регулярность | Посты выходят в запланированное время |
| Охват / просмотры | Растут месяц к месяцу |
| Переходы на сайт / мини-апп | > X в месяц |
| Новые подписчики | > X в месяц |
| Время основателя на контент | < 2 часов в неделю |

## 4. ГДЕ МОЖНО СЛОМАТЬСЯ?

### Проблемы планирования
| Ошибка | Последствие |
|--------|------------|
| Нет контент-плана → темы придумываются на ходу | Хаос, пропуски, слабые тексты |
| Бриф слишком общий ("пост про управление") | Пост получается ни о чём |
| Темы не связаны с продуктом и воронкой | Подписчики есть, продаж нет |

### Проблемы создания
| Ошибка | Последствие |
|--------|------------|
| Нет времени → пост пишется за 10 мин | Низкое качество, мало реакций |
| Текст для Telegram = текст для ВК | ВК алгоритмы не продвигают |
| Нет визуала | Посты хуже заходят в ВК |
| Голос бренда плавает | Аудитория не узнаёт стиль |

### Проблемы публикации
| Ошибка | Последствие |
|--------|------------|
| Пост не запланирован → выходит поздно ночью | Охват падает |
| Планировщик не сработал | Пост не вышел, никто не заметил |
| Пост вышел с опечаткой | Приходится удалять / исправлять |

### Проблемы мониторинга
| Ошибка | Последствие |
|--------|------------|
| Статистика не фиксируется | Непонятно, что работает |
| Нет ответов на комментарии | Аудитория перестаёт писать |

## 5. ПРЕДЛОЖЕНИЕ ПО АВТОМАТИЗАЦИИ

### Что автоматизируем, что оставляем

| Шаг | Статус | Решение |
|-----|--------|---------|
| Выбор тем | Ручной | Основатель выбирает темы 1 раз в месяц |
| Заполнение плана | Ручной → Авто | Google Sheets с шаблоном + форма заполнения |
| Написание черновика | Авто | Claude API генерирует текст по брифу |
| Редактура | Ручной | Основатель правит черновик (5–10 мин) |
| Визуал | Полуавто | Шаблоны Canva с переменными (подставить заголовок) |
| Адаптация для ВК | Авто | Claude переписывает Telegram-пост под ВК |
| Планирование публикации | Авто | SMMPlanner / Telegram Bot API |
| Мониторинг | Полуавто | Статистика → Google Sheets (ручной ввод или API) |
| Анализ | Авто | Google Sheets считает лучшие посты автоматически |

---

### Схема автоматизации

```
[Google Sheets — Контент-план]
        ↓
  Основатель заполняет бриф
  (тема, боль, CTA, дата)
        ↓
[Claude API]
  Генерирует черновик по брифу
  + адаптирует для ВК
        ↓
  Основатель проверяет (5–10 мин)
  → правит в Google Sheets
        ↓
[Telegram Bot]
  Читает одобренный пост из таблицы
  Публикует в канал в нужное время
        ↓
[Опционально: SMMPlanner / ВК API]
  Публикует в ВК
        ↓
[Google Sheets — Статистика]
  Фиксируются просмотры, реакции
```

---

### Инструменты

| Инструмент | Для чего | Стоимость |
|-----------|---------|-----------|
| **Google Sheets** | Контент-план, статистика, брифы | Бесплатно |
| **Claude API** | Генерация черновиков по брифу, адаптация для ВК | ~$5–20/мес |
| **Telegram Bot API** | Публикация постов по расписанию | Бесплатно |
| **Canva** | Визуалы по шаблонам | Бесплатно / $13/мес Pro |
| **SMMPlanner** | Планирование в ВК | ~700 руб/мес |

---

### Что даёт автоматизация

| Было | Стало |
|------|-------|
| 3–7 часов в неделю на контент | ~1–1.5 часа (только правка и визуал) |
| Пишу с нуля каждый пост | Правлю готовый черновик |
| Публикую вручную | Бот публикует сам |
| ВК адаптирую вручную | Claude переписывает автоматически |
| Статистика нигде | Всё в одной таблице |

---

### Минимальный стек для старта (MVP)

1. **Google Sheets** — контент-план с колонками:
   `Дата | Рубрика | Тема | Боль | CTA | Черновик TG | Черновик ВК | Статус | Просмотры`

2. **Claude API** — скрипт (Apps Script в Google Sheets):
   - Читает строку с брифом
   - Отправляет запрос в Claude
   - Записывает черновик обратно в таблицу

3. **Telegram Bot** — скрипт на Node.js / Python:
   - Раз в день проверяет таблицу
   - Если статус = "Одобрено" и дата = сегодня → публикует

**Время на MVP:** 4–8 часов разработки
**Результат:** Контент-конвейер, который работает без постоянного участия основателя
