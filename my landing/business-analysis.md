# Анализ бизнес-процессов и кандидаты на автоматизацию

**Бизнес:** Платформа управления бизнесом / Диагностика индекса управляемости
**Каналы:** Telegram (канал + мини-апп), сайт, ВКонтакте (планируется)
**Дата:** 2026-03-19

---

## 1. СХЕМА БИЗНЕСА

```
[1. Первое касание]
       ↓
[2. Переход на платформу]
       ↓
[3. Диагностика]
       ↓
[4. Получение результата]
       ↓
[5. Продажа / активация]
       ↓
[6. Работа на платформе]
       ↓
[7. Удержание / upsell]
```

### Этапы подробно

| # | Этап | Что происходит |
|---|------|----------------|
| 1 | **Первое касание** | Пользователь видит пост в Telegram-канале или ВК, попадает на сайт |
| 2 | **Переход на платформу** | Кликает на ссылку, открывает мини-апп или сайт, регистрируется |
| 3 | **Диагностика** | Проходит тест — отвечает на вопросы по зонам управляемости |
| 4 | **Получение результата** | Видит индекс, провалы по зонам, персональные выводы |
| 5 | **Продажа / активация** | Получает оффер, принимает решение, оплачивает / активирует доступ |
| 6 | **Работа на платформе** | Использует инструменты для закрытия слабых мест |
| 7 | **Удержание / upsell** | Получает напоминания, видит прогресс, докупает модули |

## 2. АНАЛИЗ ПРОЦЕССОВ — ЧТО ДЕЛАЕТСЯ ВРУЧНУЮ

### Этап 1 — Первое касание (контент)
| Действие | Кто делает | Время |
|----------|-----------|-------|

| Написать пост для ВК | То же | 20–40 мин/пост |
| Сделать картинку / обложку | Основатель / дизайнер | 20–40 мин |
| Опубликовать / поставить в расписание | Вручную | 5–10 мин |
| **Итого в неделю (2–3 поста):** | | **2–5 часов** |

| Действие | Кто делает | Время |
|----------|-----------|-------|
| Ответить на вопросы в директе Telegram | Основатель | 10–20 мин/диалог |
| Ответить на комментарии в ВК | То же | 10–15 мин |
| Напомнить о незавершённой регистрации | Вручную или никак | — |
| **Итого в неделю:** | | **1–3 часа** |

### Этап 3 — Диагностика
| Действие | Кто делает | Время |
|----------|-----------|-------|
| Если мини-апп работает — автоматически | Система | 0 |
| Помочь пользователю, если застрял | Основатель | 10–20 мин |
| Собрать результаты вручную (если нет БД) | Основатель | 20–30 мин |

| Действие | Кто делает | Время |

| Написать персональный комментарий | Основатель | 10–20 мин |

| Действие | Кто делает | Время |

| Действие | Кто делает | Время |
|----------|-----------|-------|

## 3. КАНДИДАТЫ НА АВТОМАТИЗАЦИЮ (топ-5)

### 🥇 #1 — Воронка после диагностики (welcome + оффер)
**Что:** После завершения теста пользователь сам получает результат и оффер — без участия основателя.
**Сейчас:** Ручная отправка результата + оффер вручную = 40–70 мин на человека.
**Как автоматизировать:** Telegram-бот отправляет результат, краткий разбор и оффер через 1–2 минуты после завершения теста. Серия из 2–3 сообщений с интервалом.
**Повторяемость:** Каждый новый пользователь.
**Экономия:** 40–70 мин на каждого лида.

---

### 🥈 #2 — Контент-план и публикация постов
**Что:** Посты для Telegram и ВК публикуются по расписанию без ручного выхода в канал.
**Сейчас:** 2–5 часов в неделю на написание + ручная публикация.
**Как автоматизировать:** Составить контент-план на месяц (1 раз), написать посты пачкой, поставить в SMMPlanner / Combot / Telemetr — дальше выходят автоматически. Дополнительно — шаблоны типовых постов.
**Повторяемость:** 2–3 поста в неделю.
**Экономия:** 1–2 часа в неделю.

---

### 🥉 #3 — Онбординг после оплаты
**Что:** Новый клиент сразу получает серию сообщений — что делать, с чего начать, где что находится.
**Сейчас:** Основатель вручную объясняет каждому.
**Как автоматизировать:** Telegram-бот запускает цепочку при получении статуса "оплачено" — 3–5 сообщений за первые 48 часов.
**Повторяемость:** Каждый новый клиент.
**Экономия:** 20–40 мин на клиента + снижение нагрузки на поддержку.

---

### #4 — Дожим незавершённых (реактивация)
**Что:** Пользователи, которые начали диагностику, но не закончили — или не купили после результата — получают 1–2 напоминания автоматически.
**Сейчас:** Никто не отслеживает, потери скрытые.
**Как автоматизировать:** Бот проверяет статус через 24 и 72 часа. Если не завершил — напоминает. Если не купил — отправляет дожимное сообщение с другим углом.
**Повторяемость:** Постоянно по мере роста базы.
**Экономия:** Конверсия вырастет без дополнительного ручного труда.

---

### #5 — Сбор обратной связи и NPS
**Что:** Через 7–14 дней после старта клиент автоматически получает 2–3 вопроса о платформе.
**Сейчас:** Не делается или вручную от случая к случаю.
**Как автоматизировать:** Бот отправляет короткий опрос через заданное время. Ответы копятся в таблице.
**Повторяемость:** Каждый клиент через N дней.
**Ценность:** Данные для улучшения продукта без отдельных усилий.

## 4. ПРИОРИТЕТЫ — ЧТО АВТОМАТИЗИРОВАТЬ ПЕРВЫМ

| Приоритет | Процесс | Усилие | Эффект | Почему сейчас |
|-----------|---------|--------|--------|---------------|
| **1** | Воронка после диагностики | Среднее | Очень высокий | Прямо влияет на конверсию в деньги. Каждый лид — потеря 40–70 мин. |
| **2** | Онбординг после оплаты | Низкое | Высокий | Один раз написать цепочку — работает вечно. Клиент не чувствует себя брошенным. |
| **3** | Контент-план и планировщик | Низкое | Средний | Освобождает голову и время. Легко внедрить уже сегодня. |
| **4** | Реактивация незавершённых | Среднее | Высокий | Скрытые деньги в существующей базе. Но требует настройки логики. |
| **5** | Сбор обратной связи | Низкое | Средний | Полезно, но некритично на старте. Делать после первых трёх. |

## ИТОГ

**Главный вывод:** Сейчас основатель — ручной бот. Каждый новый пользователь требует 1–2 часа живого времени. Это не масштабируется.

**Первый шаг:** Настроить Telegram-бота с тремя сценариями:
1. После завершения диагностики → автоматический результат + оффер
2. После оплаты → серия онбординг-сообщений
3. После 24ч без действия → напоминание

Это закрывает приоритеты #1, #2 и #4 одним инструментом.
