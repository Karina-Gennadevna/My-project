# Правила работы с проектом

## Структура файлов
- Каждая страница — один самодостаточный HTML-файл. Всё внутри: стили, скрипты, разметка.
- Внешние библиотеки (Bootstrap, Tailwind CDN, jQuery и т.д.) не используем. Только чистые HTML, CSS, JS.
- Шрифты с Google Fonts подключать можно.

## Дизайн
- Фон: тёмный (оттенки `#0D0F11` — `#1A1A1A`).
- Текст: светлый (`#F1F5F9`, `#E2E8F0`), приглушённый (`#94A3B8`).
- Акцентный цвет: **оранжевый** (`#F97316` и близкие оттенки).
- Никаких ярких фонов, пёстрых градиентов, лишних украшений.

## Адаптивность
- Все страницы обязаны корректно отображаться на мобильных (от 320px).
- Верстать через flexbox / grid. Не использовать фиксированные пиксельные ширины для блоков.
- Всегда добавлять `<meta name="viewport" content="width=device-width, initial-scale=1.0">`.

## Текст на страницах
- Простой и живой язык, без канцелярита и корпоративного новояза.
- Никаких «осуществляем», «реализуем», «в рамках». Пишем как говорим.
- Коротко: одна мысль — одно предложение.

## Перед созданием нового файла
- Сначала описать: что за страница, какие блоки, какой порядок.
- Получить подтверждение — и только потом писать код.
